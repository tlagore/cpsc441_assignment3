Running the server:
==============
Use the following command

    java -jar ffserver.jar

to start the server. To stop the server, type "quit" on the screen.

Using the above command, the server starts with a set of default parameters.
You can run the server in the following modes to specify your own parameters.

1) Non-verbose Mode:

    java -jar ffserver.jar port delay loss

where,

port: the server local port number.

delay: the average ACK delay.

The server uses this parameter to wait for some random delay before
sending each ACK. The actual delay is generated randomly so that the
average delay is equal to this parameter.


loss: the segment loss probability.

It specifies the ratio of lost segments at the server. The server randomly
drops arriving segments. The probability that the server drops a segment
is given by this parameter.
 

2) Verbose Mode:

    java -jar ffserver.jar -v port delay loss
 

By specifying the option '-v', the server works in verbose mode, where
it prints detailed information regarding received segments and transmitted
ACKs.

 

Running the client
==============
Use the following command

    java -jar fastftp.jar file

where, file specifies the file name to be transferred to the server. If it is desired to
specify other options as well, then use the full command as follows:

    java -jar fastftp.jar server port file window timeout

where,

server: server name.

port: server port number.

file: file to be trasnferred.

window: the window size (in segments)

timeout: value of timeout for the retransmission timer (in milli-seconds)
